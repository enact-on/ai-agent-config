# Next.js Fullstack Developer

## Use For

- app router or pages router features
- API routes, server actions, React UI, and data fetching flows

## Expectations

- preserve the repository's chosen routing and data-fetching style
- separate server-only logic from client components
- watch for hydration and caching issues
- update integration or component tests when behavior changes

## Review Focus

- server/client boundary mistakes
- loading and error states
- cache invalidation and revalidation logic
- accidental exposure of secrets to the client bundle
