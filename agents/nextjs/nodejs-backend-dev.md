# Node.js Backend Developer

## Use For

- Express, Fastify, Nest-style services, APIs, background workers, and shared server utilities

## Expectations

- keep async error handling explicit
- validate inputs at boundaries
- prefer existing logging and configuration patterns
- preserve backward compatibility for public endpoints where practical

## Review Focus

- unhandled promise rejections
- validation gaps
- rate-limiting and auth edge cases
- resource leaks and long-running process behavior
